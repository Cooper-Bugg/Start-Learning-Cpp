```markdown
# portfolio-apps

Collection of polishable example applications intended to be turned into portfolio pieces. These apps are more feature-rich and require larger, more opinionated designs than the smaller exercises in earlier phases.

Example ideas:
- Fly-fishing log: record outings, GPS coordinates, catches, and exportable summaries.
- Strategy-roulette: small game simulation engine that generates and evaluates strategy permutations.
- Deck-builder backend: card collection and deck building logic with import/export and basic simulation of match outcomes.

Goals & expectations:
- Each project in this folder should be treated as an independent repo when it grows beyond a prototype.
- Provide a polished `README.md`, usage examples, and at least one small end-to-end test or sample dataset.

Build & run (guidance):
- For small apps, provide a `CMakeLists.txt` or simple `Makefile` and a `main.cpp` demonstrating common workflows.
- Include example input files and a screenshot or sample output in the README to help reviewers quickly understand the project.

Notes:
- When turning these into real portfolio items, consider adding packaging, CLI docs, and a short demo video or GIF.
```
